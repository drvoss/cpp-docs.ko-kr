## <a name="remarks"></a>설명

OLE 항목은 서버 응용 프로그램에서 만들고 유지 관리하는 데이터를 나타내며, 이는 사용자에게 단일 문서로 표시되도록 문서에 "원활하게" 통합될 수 있습니다. 그 결과 OLE 항목과 포함된 문서로 구성된 "복합 문서"가 생성됩니다.

OLE 항목은 포함되거나 링크될 수 있습니다. 포함된 경우 해당 데이터는 복합 문서의 일부로 저장됩니다. 연결된 경우 해당 데이터는 서버 응용 프로그램에서 만든 별도의 파일의 일부로 저장되며 해당 파일에 대한 링크만 복합 문서에 저장됩니다. 모든 OLE 항목에는 편집하기 위해 호출해야 하는 서버 응용 프로그램을 지정하는 정보가 포함되어 있습니다.

`COleClientItem`서버 응용 프로그램의 요청에 대한 응답으로 호출되는 여러 재정의 가능한 함수를 정의합니다. 이러한 재정의 가능한 일반적으로 알림 역할을 합니다. 이렇게 하면 서버 응용 프로그램이 OLE 항목을 편집할 때 사용자가 변경한 내용을 컨테이너에 알리거나 편집 하는 동안 필요한 정보를 검색할 수 있습니다.

`COleClientItem`[COleDocument,](../../mfc/reference/coledocument-class.md) [COleLinkingDoc](../../mfc/reference/colelinkingdoc-class.md)또는 [COleServerDoc](../../mfc/reference/coleserverdoc-class.md) 클래스와 함께 사용할 수 있습니다. 을 `COleClientItem`사용하려면 클래스를 파생하고 [컨테이너가](#onchange) 항목에 대한 변경 사항에 응답하는 방법을 정의하는 OnChange 멤버 함수를 구현합니다. 내부 활성화를 지원하려면 [OnGetItemPosition](#ongetitemposition) 멤버 함수를 재정의합니다. 이 함수는 OLE 항목의 표시된 위치에 대한 정보를 제공합니다.

컨테이너 인터페이스 사용에 대한 자세한 내용은 [컨테이너 구현](../../mfc/containers-implementing-a-container.md) 및 [활성화](../../mfc/activation-cpp.md)문서를 참조하십시오.

> [!NOTE]
> Windows SDK는 포함된 항목과 연결된 항목을 "개체"로 참조하며 항목 유형을 "클래스"라고 합니다. 이 참조는 "item"이라는 용어를 사용하여 OLE 엔터티를 해당 C++ 개체와 구분하고 "type"이라는 용어를 사용하여 OLE 범주를 C++ 클래스와 구분합니다.

## <a name="coleclientitemactivate"></a><a name="activate"></a>COle클라이언트항목::활성화

예외가 throw될 때 사용자 고유의 처리를 수행할 수 있도록 [DoVerb](#doverb) 대신 지정된 동사를 실행하려면 이 함수를 호출합니다.

```cpp
void Activate(
    LONG nVerb,
    CView* pView,
    LPMSG lpMsg = NULL);
```

### <a name="parameters"></a>매개 변수

*nVerb*<br/>
실행할 동사를 지정합니다. 다음 중 하나일 수 있습니다.

|값|의미|기호|
|-----------|-------------|------------|
|- 0|기본 동사|OLEIVERB_PRIMARY|
|- 1|보조 동사|(없음)|
|- 1|편집할 항목 표시|OLEIVERB_SHOW|
|- 2|별도의 창에서 항목 편집|OLEIVERB_OPEN|
|- 3|항목 숨기기|OLEIVERB_HIDE|

-1 값은 일반적으로 다른 동사의 별칭입니다. 열린 편집이 지원되지 않으면 -2는 -1과 동일한 효과를 가짐을 가합니다. 추가 값은 Windows SDK에서 [IOleObject::Doverb를](/windows/win32/api/oleidl/nf-oleidl-ioleobject-doverb) 참조하십시오.

*pView*<br/>
OLE 항목을 포함하는 컨테이너 뷰 창에 대한 포인터; 서버 응용 프로그램에서 내부 활성화에 사용됩니다. 컨테이너가 내부 활성화를 지원하지 않는 경우 이 매개 변수는 NULL이어야 합니다.

*lpMsg*<br/>
항목을 활성화한 메시지에 대한 포인터입니다.

### <a name="remarks"></a>설명

Microsoft Foundation 클래스 라이브러리를 사용하여 서버 응용 프로그램을 작성한 경우 이 함수를 `COleServerItem` 사용하면 해당 개체의 [OnDoVerb](../../mfc/reference/coleserveritem-class.md#ondoverb) 멤버 함수가 실행됩니다.

기본 동사가 편집이고 *nVerb* 매개 변수에 0이 지정되면 OLE 항목을 편집할 수 있도록 서버 응용 프로그램이 시작됩니다. 컨테이너 응용 프로그램이 내부 활성화를 지원하는 경우 편집을 수행할 수 있습니다. 컨테이너가 내부 활성화를 지원하지 않는 경우(또는 열기 동사가 지정된 경우) 서버가 별도의 창에서 시작되고 편집을 수행할 수 있습니다. 일반적으로 컨테이너 응용 프로그램의 사용자가 OLE 항목을 두 번 클릭하면 *nVerb* 매개 변수의 기본 동사에 대한 값이 사용자가 취할 수 있는 작업을 결정합니다. 그러나 서버가 하나의 작업만 지원하는 경우 *nVerb* 매개 변수에 지정된 값에 관계없이 해당 작업을 수행합니다.

자세한 내용은 Windows SDK의 [IOleObject::Doverb를](/windows/win32/api/oleidl/nf-oleidl-ioleobject-doverb) 참조하십시오.

- 윈도우 NT 드래그 지연 시간은 HKEY_LOCAL_MACHINE 저장됩니다 \SOFTWARE\마이크로 소프트 \윈도우 \NT \현재버전\IniFileMapping\win.ini\Windows\DragDelay.

## <a name="coleclientitemdoverb"></a><a name="doverb"></a>COle클라이언트항목::D

호출하여 `DoVerb` 지정된 동사를 실행합니다.

```
virtual BOOL DoVerb(
    LONG nVerb,
    CView* pView,
    LPMSG lpMsg = NULL);
```

### <a name="parameters"></a>매개 변수

*nVerb*<br/>
실행할 동사를 지정합니다. 다음 중 하나를 포함할 수 있습니다.

|값|의미|기호|
|-----------|-------------|------------|
|- 0|기본 동사|OLEIVERB_PRIMARY|
|- 1|보조 동사|(없음)|
|- 1|편집할 항목 표시|OLEIVERB_SHOW|
|- 2|별도의 창에서 항목 편집|OLEIVERB_OPEN|
|- 3|항목 숨기기|OLEIVERB_HIDE|

-1 값은 일반적으로 다른 동사의 별칭입니다. 열린 편집이 지원되지 않으면 -2는 -1과 동일한 효과를 가짐을 가합니다. 추가 값은 Windows SDK에서 [IOleObject::Doverb를](/windows/win32/api/oleidl/nf-oleidl-ioleobject-doverb) 참조하십시오.

*pView*<br/>
뷰 창에 대한 포인터; 이 문제는 서버에서 내부 활성화에 사용됩니다. 컨테이너 응용 프로그램이 내부 활성화를 허용하지 않는 경우 이 매개 변수는 NULL이어야 합니다.

*lpMsg*<br/>
항목을 활성화한 메시지에 대한 포인터입니다.

### <a name="return-value"></a>Return Value

동사가 성공적으로 실행된 경우 0이 아닙니다. 그렇지 않으면 0.

### <a name="remarks"></a>설명

이 함수는 [활성화](#activate) 멤버 함수를 호출하여 동사를 실행합니다. 또한 예외를 catch하고 메시지를 throw하면 사용자에게 메시지 상자를 표시합니다.

기본 동사가 편집이고 *nVerb* 매개 변수에 0이 지정되면 OLE 항목을 편집할 수 있도록 서버 응용 프로그램이 시작됩니다. 컨테이너 응용 프로그램이 내부 활성화를 지원하는 경우 편집을 수행할 수 있습니다. 컨테이너가 내부 활성화를 지원하지 않는 경우(또는 열기 동사가 지정된 경우) 서버가 별도의 창에서 시작되고 편집을 수행할 수 있습니다. 일반적으로 컨테이너 응용 프로그램의 사용자가 OLE 항목을 두 번 클릭하면 *nVerb* 매개 변수의 기본 동사에 대한 값이 사용자가 취할 수 있는 작업을 결정합니다. 그러나 서버가 하나의 작업만 지원하는 경우 *nVerb* 매개 변수에 지정된 값에 관계없이 해당 작업을 수행합니다.

*rString*<br/>
OLE 항목의 형식을 설명하는 문자열이 반환되는 [CString](../../atl-mfc-shared/reference/cstringt-class.md) 개체에 대한 참조입니다.

## <a name="coleclientitemonchange"></a><a name="onchange"></a>COle클라이언트항목::온체인지

사용자가 OLE 항목을 수정, 저장 또는 닫을 때 프레임워크에서 호출됩니다.

```
virtual void OnChange(
    OLE_NOTIFICATION nCode,
    DWORD dwParam);
```

### <a name="parameters"></a>매개 변수

*nCode*<br/>
서버가 이 항목을 변경한 이유입니다. 다음 값 중 하나일 수 있습니다.

- OLE_CHANGED OLE 항목의 모양이 변경되었습니다.

- OLE_SAVED 항목이 저장되었습니다.

- OLE_CLOSED OLE 항목이 닫혔습니다.

- OLE_CHANGED_STATE OLE 항목이 한 상태에서 다른 상태로 변경되었습니다.

*dwParam*<br/>
*nCode가* OLE_SAVED OLE_CLOSED 경우 이 매개 변수는 사용되지 않습니다. *nCode가* OLE_CHANGED 경우 이 매개 변수는 변경된 OLE 항목의 측면을 지정합니다. 가능한 값은 [COleClientItem::Draw의](#draw) *dwParam* 매개 변수를 참조하십시오. *nCode가* OLE_CHANGED_STATE 경우 이 `COleClientItem::ItemState` 매개 변수는 예명된 값이며 입력되는 상태를 설명합니다. 다음 `emptyState`값 `loadedState` `openState` `activeState`중 하나를 가질 수 있습니다. `activeUIState`

### <a name="remarks"></a>설명

서버 응용 프로그램이 Microsoft Foundation 클래스 라이브러리를 사용하여 작성된 경우 이 `Notify` 함수는 `COleServerDoc` 또는 `COleServerItem`.) 기본 구현은 *nCode가* OLE_CHANGED 또는 OLE_SAVED 경우 컨테이너 문서를 수정된 것으로 표시합니다.

OLE_CHANGED_STATE 경우 [GetItemState에서](#getitemstate) 반환된 현재 상태는 여전히 이전 상태이므로 이 상태가 변경되기 전에 현재 상태였던 상태가 됩니다.

이 함수를 재정의하여 OLE 항목의 상태의 변경 사항에 응답합니다. 일반적으로 항목이 표시되는 영역을 무효화하여 항목의 모양을 업데이트합니다. 재정의를 시작할 때 기본 클래스 구현을 호출합니다.

### <a name="parameters"></a>매개 변수

*ppDocFrame*<br/>

기본 구현이 응용 프로그램에 맞지 않는 경우에만 이 함수를 재정의합니다. 예를 들어 응용 프로그램에 SDI 또는 MDI와 다른 사용자 인터페이스 패러다임이 있는 경우입니다. 이 고급 재정의 할 수 있습니다.

자세한 내용은 [IOleInPlaceSite::GetWindowContext](/windows/win32/api/oleidl/nf-oleidl-ioleinplacesite-getwindowcontext) 및 Windows SDK의 [올레인플레이스프레임정보](/windows/win32/api/oleidl/ns-oleidl-oleinplaceframeinfo) 구조를 참조하십시오.

## <a name="coleclientitemoninsertmenus"></a><a name="oninsertmenus"></a>COle클라이언트항목::온인서인스메뉴

### <a name="parameters"></a>매개 변수

*lpMenuWidths*<br/>
파일, 편집, 컨테이너, 개체, 창, 도움말 : 다음 각 메뉴 그룹에 있는 메뉴 수를 나타내는 여섯 LONG 값의 배열을 가리킵니다. 컨테이너 응용 프로그램은 이 배열의 요소 0, 2 및 4에 해당하는 파일, 컨테이너 및 창 메뉴 그룹을 담당합니다.

### <a name="remarks"></a>설명

그런 다음 이 메뉴는 서버로 전달되어 자체 메뉴를 삽입하여 복합 메뉴를 만듭니다. 이 함수는 여러 복합 메뉴를 빌드하기 위해 반복적으로 호출할 수 있습니다.

기본 구현은 *pMenu에* 삽입하여 현재 위치 컨테이너 메뉴를 공유합니다. 즉, 파일, 컨테이너 및 창 메뉴 그룹입니다. [CDocTemplate::SetContainerInfo는이](../../mfc/reference/cdoctemplate-class.md#setcontainerinfo) 메뉴 리소스를 설정 하는 데 사용 됩니다. 또한 기본 구현은 메뉴 리소스에 따라 *lpMenuWidths의*요소 0, 2 및 4에 적절한 값을 할당합니다. 기본 구현이 응용 프로그램에 적합하지 않은 경우 이 함수를 재정의합니다. 예를 들어 응용 프로그램에서 리소스를 문서 형식과 연결하기 위해 문서 템플릿을 사용하지 않는 경우를 예로 들 수 있습니다. 이 함수를 재정의하는 경우 [OnSetMenu](#onsetmenu) 및 [OnRemoveMenus도](#onremovemenus)재정의해야 합니다. 이 고급 재정의 할 수 있습니다.

자세한 내용은 [IOleInPlaceFrame::Windows](/windows/win32/api/oleidl/nf-oleidl-ioleinplaceframe-insertmenus) SDK의 삽입메뉴를 참조하십시오.

OLE 항목이 로드된 상태로 돌아갈 때 저장되는 상황을 지정하는 플래그입니다. 가능한 값 목록은 [COleClientItem::Close](#close)를 참조하십시오.

### <a name="remarks"></a>설명

`Release`소멸자에서 `COleClientItem` 호출됩니다.

자세한 내용은 Windows SDK에서 [IUnknown::Release를](/windows/win32/api/unknwn/nf-unknwn-iunknown-release) 참조하십시오.


### <a name="remarks"></a>설명

## <a name="coleclientitemrun"></a><a name="run"></a>COle클라이언트항목::실행

이 항목과 연결된 응용 프로그램을 실행합니다.

```cpp
void Run();
```

### <a name="remarks"></a>설명

항목을 `Run` 활성화하기 전에 멤버 함수를 호출하여 서버 응용 프로그램을 시작합니다. 이 작업은 [Activate](#activate) 및 [DoVerb에](#doverb)의해 자동으로 수행되므로 일반적으로 이 함수를 호출할 필요가 없습니다. [DoVerb를](#doverb)실행하기 전에 [SetExtent](#setextent)와 같은 항목 특성을 설정하기 위해 서버를 실행해야 하는 경우 이 함수를 호출합니다.

## <a name="coleclientitemsetdrawaspect"></a><a name="setdrawaspect"></a>COle클라이언트항목::세트드로우측면

멤버 `SetDrawAspect` 함수를 호출하여 항목의 "종횡비 또는 보기"를 설정합니다.

```
virtual void SetDrawAspect(DVASPECT nDrawAspect);
```

### <a name="parameters"></a>매개 변수

*nDrawAspect*<br/>
DVASPECT 열거형의 값입니다. 이 매개 변수는 다음 값 중 하나를 가질 수 있습니다.

- DVASPECT_CONTENT 항목은 컨테이너 내부에 포함된 개체로 표시될 수 있는 방식으로 표시됩니다.

- DVASPECT_THUMBNAIL 항목은 검색 도구에 표시될 수 있도록 "썸네일" 표현으로 렌더링됩니다.

- DVASPECT_ICON 항목은 아이콘으로 표시됩니다.

- DVASPECT_DOCPRINT 항목은 파일 메뉴에서 인쇄 명령을 사용하여 인쇄된 것처럼 표시됩니다.

### <a name="remarks"></a>설명

이 종횡주는 해당 함수의 *nDrawAspect* 인수에 대한 기본값을 사용할 때 [Draw에](#draw) 의해 항목을 렌더링하는 방법을 지정합니다.

이 함수는 사용자가 요청할 때 아이콘 표시 측면을 활성화하기 위해 변경 아이콘(및 변경 아이콘 대화 상자를 직접 호출하는 기타 대화 상자)에 의해 자동으로 호출됩니다.

## <a name="coleclientitemsetextent"></a><a name="setextent"></a>COle클라이언트항목::세트익

이 함수를 호출하여 OLE 항목에 사용할 수 있는 공간의 양을 지정합니다.

```cpp
void SetExtent(
    const CSize& size,
    DVASPECT nDrawAspect = DVASPECT_CONTENT);
```

### <a name="parameters"></a>매개 변수

*size*<br/>
크기 정보가 포함된 [CSize](../../atl-mfc-shared/reference/csize-class.md) 개체입니다.

*nDrawAspect*<br/>
경계를 설정할 OLE 항목의 측면을 지정합니다. 가능한 값은 [SetDrawAspect](#setdrawaspect)를 참조하십시오.

### <a name="remarks"></a>설명

Microsoft Foundation 클래스 라이브러리를 사용하여 서버 응용 프로그램을 작성한 경우 해당 `COleServerItem` 개체의 [OnSetExtent](../../mfc/reference/coleserveritem-class.md#onsetextent) 멤버 함수가 호출됩니다. 그런 다음 OLE 항목은 그에 따라 디스플레이를 조정할 수 있습니다. 치수는 MM_HIMETRIC 단위여야 합니다. 사용자가 OLE 항목의 크기를 조정하거나 레이아웃 협상을 지원하는 경우 이 함수를 호출합니다.

자세한 내용은 [IOleObject::Set익](/windows/win32/api/oleidl/nf-oleidl-ioleobject-setextent) Windows SDK를 참조하십시오.

## <a name="coleclientitemsethostnames"></a><a name="sethostnames"></a>COle클라이언트항목::세트호스트 이름

이 함수를 호출하여 컨테이너 응용 프로그램의 이름과 포함된 OLE 항목에 대한 컨테이너 이름을 지정합니다.

```cpp
void SetHostNames(
    LPCTSTR lpszHost,
    LPCTSTR lpszHostObj);
```

### <a name="parameters"></a>매개 변수

*lpszHost*<br/>
컨테이너 응용 프로그램의 사용자가 볼 수 있는 이름에 대한 포인터입니다.

*lpszHostObj*<br/>
OLE 항목을 포함하는 컨테이너의 식별 문자열을 포인터합니다.

### <a name="remarks"></a>설명

서버 응용 프로그램이 MFC 라이브러리를 사용하여 작성된 경우 이 함수는 OLE 항목을 포함하는 `COleServerDoc`문서의 [OnSetHostNames](../../mfc/reference/coleserverdoc-class.md#onsethostnames) 멤버 함수를 호출합니다. 이 정보는 OLE 항목을 편집할 때 창 제목에 사용됩니다. 컨테이너 문서가 로드될 때마다 프레임워크는 문서의 모든 OLE 항목에 대해 이 함수를 호출합니다. `SetHostNames`포함된 항목에만 적용됩니다. 편집을 위해 포함된 OLE 항목이 활성화될 때마다 이 함수를 호출할 필요가 없습니다.

또한 개체가 로드되거나 파일이 다른 이름으로 저장될 때 응용 프로그램 이름과 문서 이름으로 자동으로 호출됩니다. 따라서 일반적으로 이 함수를 직접 호출할 필요는 없습니다.

자세한 내용은 Windows SDK의 [IOleObject::SetHostNames를](/windows/win32/api/oleidl/nf-oleidl-ioleobject-sethostnames) 참조하십시오.

## <a name="coleclientitemsetitemrects"></a><a name="setitemrects"></a>COle클라이언트항목::세트아이템 렉트

이 함수를 호출하여 경계 사각형 또는 OLE 항목의 표시되는 사각형을 설정합니다.

```
BOOL SetItemRects(
    LPCRECT lpPosRect = NULL,
    LPCRECT lpClipRect = NULL);
```

### <a name="parameters"></a>매개 변수

*lprcPosRect*<br/>
클라이언트 좌표에서 부모 창을 기준으로 OLE 항목의 경계를 포함하는 사각형에 대한 포인터입니다.

*lprcClipRect*<br/>
클라이언트 좌표에서 부모 창을 기준으로 OLE 항목의 표시되는 부분의 경계를 포함하는 사각형에 대한 포인터입니다.

자세한 내용은 [IOleInPlaceObject::SetObjectRects](/windows/win32/api/oleidl/nf-oleidl-ioleinplaceobject-setobjectrects) Windows SDK를 참조하십시오.

## <a name="coleclientitemsetlinkupdateoptions"></a><a name="setlinkupdateoptions"></a>COle클라이언트항목::SetLinkUpdate옵션

이 함수를 호출하여 지정된 링크된 항목의 표시에 대한 링크 업데이트 옵션을 설정합니다.

```cpp
void SetLinkUpdateOptions(OLEUPDATE dwUpdateOpt);
```

### <a name="parameters"></a>매개 변수

*dwUpdateOpt*<br/>
이 항목에 대한 링크 업데이트 옵션의 값입니다. 이 값은 다음 중 하나여야 합니다.

- OLEUPDATE_ALWAYS 가능하면 연결된 항목을 업데이트합니다. 이 옵션은 링크 대화 상자에서 자동 링크 업데이트 라디오 단추를 지원합니다.

- OLEUPDATE_ONCALL 컨테이너 응용 프로그램의 [요청(UpdateLink](#updatelink) 멤버 함수가 호출될 때)에서만 연결된 항목을 업데이트합니다. 이 옵션은 링크 대화 상자의 수동 링크 업데이트 라디오 단추를 지원합니다.

### <a name="remarks"></a>설명

일반적으로 링크 대화 상자에서 사용자가 선택한 업데이트 옵션을 변경하면 안 됩니다.

자세한 내용은 [IOleLink::SetUpdateWindowsSK에서](/windows/win32/api/oleidl/nf-oleidl-iolelink-setupdateoptions) 참조하세요.

## <a name="coleclientitemsetprintdevice"></a><a name="setprintdevice"></a>COle클라이언트항목::세트프린트장치

이 함수를 호출하여 이 항목의 인쇄 대상 장치를 변경합니다.

```
BOOL SetPrintDevice(const DVTARGETDEVICE* ptd);
BOOL SetPrintDevice(const PRINTDLG* ppd);
```

### <a name="parameters"></a>매개 변수

*ptd*<br/>
새 인쇄 대상 장치에 대한 정보가 포함된 [DVTARGETDEVICE](/windows/win32/api/objidl/ns-objidl-dvtargetdevice) 데이터 구조에 대한 포인터입니다. NULL일 수 있습니다.

*ppd*<br/>
새 인쇄 대상 장치에 대한 정보가 포함된 [PRINTDLG](/windows/win32/api/commdlg/ns-commdlg-printdlga) 데이터 구조에 대한 포인터입니다. NULL일 수 있습니다.

### <a name="return-value"></a>Return Value

함수가 성공한 경우 0이 아닙니다. 그렇지 않으면 0.

### <a name="remarks"></a>설명

이 함수는 항목에 대한 인쇄 대상 장치를 업데이트하지만 프레젠테이션 캐시를 새로 고치지는 않습니다. 항목에 대한 프레젠테이션 캐시를 업데이트하려면 [UpdateLink](#updatelink)를 호출합니다.

이 함수에 대한 인수에는 OLE 시스템에서 대상 장치를 식별하는 데 사용하는 정보가 포함됩니다. 구조에는 `PRINTDLG` Windows에서 일반적인 Print 대화 상자를 초기화하는 데 사용하는 정보가 포함되어 있습니다. 사용자가 대화 상자를 닫은 후 Windows는 이 구조에서 사용자의 선택에 대한 정보를 반환합니다. [CPrintDialog](../../mfc/reference/cprintdialog-class.md) 개체의 `m_pd` 멤버는 `PRINTDLG` 구조체입니다.

이 구조에 대한 자세한 내용은 Windows SDK의 [PRINTDLG를](/windows/win32/api/commdlg/ns-commdlg-printdlga) 참조하십시오.

자세한 내용은 Windows SDK의 [DVTARGETDEVICE를](/windows/win32/api/objidl/ns-objidl-dvtargetdevice) 참조하십시오.

## <a name="coleclientitemupdatelink"></a><a name="updatelink"></a>COle클라이언트항목::업데이트링크

이 함수를 호출하여 OLE 항목의 프레젠테이션 데이터를 즉시 업데이트합니다.

```
BOOL UpdateLink();
```

### <a name="return-value"></a>Return Value

성공하면 0이 아닌 값이고, 실패하면 0입니다.

### <a name="remarks"></a>설명

연결된 항목의 경우 함수는 OLE 항목에 대한 새 프레젠테이션을 얻기 위해 링크 소스를 찾습니다. 이 프로세스에는 하나 이상의 서버 응용 프로그램을 실행하는 것이 포함될 수 있으며, 이 경우 시간이 많이 소요될 수 있습니다. 포함된 항목의 경우 함수는 재귀적으로 작동하여 포함된 항목에 오래된 링크가 포함되어 있는지 확인하고 업데이트합니다. 사용자는 링크 대화 상자를 사용하여 개별 링크를 수동으로 업데이트할 수도 있습니다.

자세한 내용은 [IOleLink::Windows](/windows/win32/api/oleidl/nf-oleidl-iolelink-update) SDK의 업데이트를 참조하십시오.
